Introduction

Seedly is a native mobile application developed for both iOS and Android. I designed Seedly as a plant care and eCommerce app for anyone passionate about gardening or looking to better care for their plants. It provides users with expert knowledge, practical tips, and a seamless shopping experience for plants, gardening tools, and essentials—all in one place. By combining plant care with eCommerce, Seedly empowers users to cultivate healthier plants and create the garden of their dreams.

This project allowed me to navigate the challenges of designing for both iOS and Android ecosystems while balancing plant care features with seamless eCommerce integration, strengthening my adaptability and problem-solving skills.

Basic App Features
  • Onboarding
  • A navigation bar that allows users to navigate through the entire application
  • User customized experienced: plant recommendations, keep track of "my plants"," get notifications of "plant care/tasks," questionaire, profile preferences
  • eCommerce Integration: Buy plants, pots, and care supplies directly from the app with secure payment options and order tracking.
  • Scan feature

Background

This is an app I designed while being a student at CareerFoundry's UI Design Program.

Research and Ideation

User Flows

Once the interviews were done, I then curated a user flow based on my users' needs and wants.

User Needs
  • When I see a plant I like, I want to be able to save it, so I can maybe add it to my collection.
  • When I add a plant to "My Plants," I want to be able to receive daily reminders of its plantcare.
  • When I see a plant I don't know of, I want to be able to use the app and scan the plant to identify its name and information.

Mid-Fidelity Wireframes

Iterations

Based on user testing, the following iterations were made to allow the user to have a more efficient experience:
  • Users wanted to see the image first as opposed to other information
  • Made image larger to give a more modern feel
  • Bookmark/Share/Rating features were moved to allow better space and focus on the main components of the recipe screen
  • Iconography was highly integrated to allow better space and eye scanning
  • Bullet points were moved to the right hand sign to focus on the ingredients first

Challenges and Solutions

This project allowed me to vastly step outside my comfort zone and become familiar with both Apple Design Resources and Material Design. As someone who was using only iOS systems my entire life, this was exceptionally challenging but thoroughly exposed me to become just as familiar with Android systems also. I would say this project challenged me the most and showcases my design quality to the highest level, which I'm incredibly proud of.

If there was anything I would've done differently in the future, it would be to constantly expose myself to various devices and operating systems, regardless if I use them or not. As this will allow me to be an exceptional designer capable of designing for any and all devices.
